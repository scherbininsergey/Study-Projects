# Таблица переходов и состояний: Страница авторизации
## https://applicant.21-school.ru/auth

---

## Описание состояний системы

| ID | Состояние | Описание |
| :--- | :--- | :--- |
| S0 | `Ожидание авторизации` | Начальное состояние перед началом взаимодействия пользователя. Отображается страница с формой «Добро пожаловать» и кнопками «Войти по Сбер ID», «Регистрация», «Войти по почте», «Я из Узбекского кампуса»  |
| S1 | `Форма входа` | Интерфейсная страница с формой «Вход по почте», содержащая поля для ввода email и пароля, кнопки «Регистрация», «Войти» и «Не помню пароль». |
| S2 | `Регистрация` | Отображается страница с формой подачи заявки на обучение с выпадающими списками «Выбери кампус», «Выбери программу», чекбоксом согласия на обработку персональных данных и кнопками «Продолжить со Сбер ID» и «Другие способы входа» |
| S3 | `Профиль пользователя` | Cтраница приложения, отображаемая после успешной аутентификации. Содержит информацию о пользователе и кнопку выхода из системы. |
| S4 | `Ошибка входа` | Уведомление об ошибке, отображаемое на форме входа при вводе неверных учетных данных. Содержит сообщение "Неверный email или пароль". |
| S5 | `Форма сброса пароля` | Интерфейсная страница для восстановления доступа, содержащая поле для ввода электронной почты. Есть кнопка «Назад» и «Отправить» |
| S6 | `Письмо отправлено` | Страница-уведомление, отображаемая после отправки запроса на сброс пароля. Сообщает пользователю, что на указанный адрес электронной почты было отправлено письмо с инструкциями. |
| S7 | `Новый пароль` | Интерфейсная страница с формой «Установите новый пароль», открывающаяся по ссылке из письма для сброса пароля. Содержит поля для ввода нового пароля и его подтверждения, а также кнопки «Отправить» и «Войти в систему». |

# Диаграмма переходов и состояний: Авторизация

```mermaid
stateDiagram-v2
    direction LR

    %% Начальное состояние
    [*] --> S0

    %% === Состояния ===
    state "S0: Главный экран" as S0
    state "S1: Вход по почте" as S1
    state "S2: Регистрация" as S2
    state "S3: Профиль (Успех)" as S3
    state "S4: Ошибка входа" as S4
    state "S5: Сброс пароля" as S5
    state "S6: Письмо отправлено" as S6
    state "S7: Новый пароль" as S7

    %% === Переходы ===
    
    %% Из S0
    S0 --> S1 : Войти по почте
    S0 --> S3 : Войти по Сбер ID
    S0 --> S2 : Регистрация

    %% Из S1
    S1 --> S3 : Успешный вход
    S1 --> S4 : Ошибка входа
    S1 --> S5 : Забыли пароль?
    S1 --> S3 : Быстрая регистрация

    %% Из S2
    S2 --> S7 : Продолжить (Сбер/Email)

    %% Из S4
    S4 --> S1 : Повторить ввод

    %% Из S5
    S5 --> S6 : Отправить ссылку
    S5 --> S0 : Назад

    %% Из S6
    S6 --> S7 : Переход из письма

    %% Из S7
    S7 --> S3 : Сохранить пароль
    S7 --> S3 : Войти

    %% Завершение
    S3 --> [*] : Выход

    %% === ВИЗУАЛЬНЫЕ РАСПОРКИ (Fix layout) ===
    %% Пустые связи помогают раздвинуть слипшиеся блоки
    S0 ~~~ S2 : " "
    S1 ~~~ S5 : " "
    S4 ~~~ S2 : " "
    S6 ~~~ S4 : " "
