# Чек-лист тестирования приложения «Найди и иди» (Black-box)

**Объект тестирования:** Мобильное приложение «Найди и иди».  
**Основание:** Техническое задание (ID SC21-01 — SC21-06).  
**Методология:** Black-box (проверка функциональности без знания внутреннего кода, основываясь на требованиях и выявленных в них пробелах).

---

## 1. Модуль поиска (SC21-01, SC21-02, SC21-03)

### Поле ввода названия магазина
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-01** | Ввод текста длиной 1 символ. | Кнопка «Поиск» неактивна (или ввод блокируется), так как мин. длина — 2 символа. |
| **BB-02** | Ввод текста длиной 2 символа. | Ввод принимается, кнопка «Поиск» становится активной (если поле единственное заполненное). |
| **BB-03** | Ввод максимально длинной строки (стресс-тест, >1000 символов). | Приложение не падает, верстка не ломается, ввод ограничивается разумным лимитом или обрабатывается корректно. |
| **BB-04** | Ввод специальных символов (`<`, `>`, `&`, `"`, `'`). | Символы отображаются как текст, скрипты не исполняются (защита от XSS), поиск работает корректно. |
| **BB-05** | Ввод только пробелов. | Система считает поле незаполненным или выдает ошибку валидации. |
| **BB-06** | Проверка отображения плейсхолдера. | Плейсхолдер виден в пустом поле. |
| **BB-07** | Исчезновение плейсхолдера при начале ввода. | Плейсхолдер исчезает сразу после ввода первого символа. |
| **BB-08** | Проверка истории поисков (автокомплит). | При фокусе на поле отображается список предыдущих запросов. |
| **BB-09** | Выбор элемента из истории поисков. | Запрос подставляется в поле, результаты поиска обновляются автоматически или после нажатия «Поиск». |

### Поле выбора типа магазина
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-10** | Открытие выпадающего меню. | Отображается список категорий (минимум: Продуктовые, Стройматериалы, Аптеки). Фраза «и т.п.» проверена на наличие других категорий в реализации. |
| **BB-11** | Проверка плейсхолдера в селекте. | До выбора отображается текст «Выберите тип магазина». |
| **BB-12** | Выбор категории из списка. | Выбранное значение отображается в поле, меню закрывается. |
| **BB-13** | Попытка сброса выбранного значения. | Проверить возможность вернуть состояние «Не выбрано» (если предусмотрено UI). |
| **BB-14** | Повторное открытие меню при уже выбранном значении. | Меню открывается, текущий выбор подсвечен. |

### Логика кнопки «Поиск»
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-15** | Оба поля пусты. | Кнопка «Поиск» неактивна (серая, не кликабельна). |
| **BB-16** | Заполнено только «Название магазина» (≥2 символов). | Кнопка «Поиск» активна. |
| **BB-17** | Заполнен только «Тип магазина». | Кнопка «Поиск» активна. |
| **BB-18** | Заполнены оба поля. | Кнопка «Поиск» активна. |
| **BB-19** | Поиск по названию несуществующего магазина. | Отображается сообщение «Ничего не найдено» или карта очищается (проверка обработки пустой выдачи). |
| **BB-20** | Поиск по типу магазина, которого нет в радиусе. | Отображается соответствующее уведомление или карта показывает пустую область. |
| **BB-21** | Отображение результатов успешного поиска. | На карте появляются флажки (маркеры) найденных объектов. |
| **BB-22** | Кластеризация результатов (при большом кол-ве). | При множестве результатов флажки группируются в кластеры (если реализовано) или список прокручивается. |

## 2. Модуль карты и маршрутизации (SC21-03, SC21-04, SC21-05)

### Взаимодействие с картой
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-23** | Скроллинг карты (жест перемещения). | Карта перемещается плавно, без задержек. |
| **BB-24** | Зуммирование карты (жест щипка). | Карта масштабируется (приближение/удаление). |
| **BB-25** | Проверка области отображения карты. | Карта занимает ровно 50% экрана (в портретной ориентации). |
| **BB-26** | Ориентация экрана (Ландшафт). | Проверить корректность отображения карты и占比 (сохраняется ли логика 50% или адаптируется под UX). |

### Кнопка «Построить маршрут» (Активация и UI)
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-27** | Активация: Найден объект + выбран флаг на карте. | Кнопка становится активной. |
| **BB-28** | Активация: Выбран только тип магазина. | Кнопка становится активной (строится до ближайшего). |
| **BB-29** | Активация: Только название введено, флаг не выбран. | Кнопка неактивна (согласно п.1 условия SC21-04). |
| **BB-30** | Визуальная проверка состояния «Активен». | Фон #fff, есть иконка (стрелка), надпись «Построить маршрут», текст контрастен. |
| **BB-31** | Определение «ближайшего» магазина. | При выборе типа строится маршрут до объекта, географически ближайшего к текущей геопозиции пользователя (проверка логики расстояния). |

### Построение и отмена маршрута
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-32** | Нажатие на кнопку «Построить маршрут». | Маршрут отображается на карте. |
| **BB-33** | Изменение состояния кнопки после нажатия. | Фон меняется на #586d73 (серый), надпись меняется на «Отменить маршрут». |
| **BB-34** | Нажатие на кнопку «Отменить маршрут». | Маршрут удаляется с карты. |
| **BB-35** | Возврат состояния кнопки после отмены. | Кнопка возвращается в исходное состояние (белый фон, надпись «Построить маршрут»). |
| **BB-36** | Попытка отмены маршрута другими способами. | Проверить, что отмена возможна **только** через кнопку (скролл карты или клик в пустоту не отменяют маршрут). |

## 3. Общие требования и сценарии использования (SC21-06)

### Сохранение состояния и Геолокация
| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-37** | Рестарт приложения с активным маршрутом. | После закрытия и повторного открытия приложения ранее построенный маршрут отображается на карте. |
| **BB-38** | Рестарт приложения без активного маршрута. | Открывается чистая карта или последний экран поиска (без навязывания старого маршрута, если он был завершен/отменен). |
| **BB-39** | Эмуляция низкого заряда батареи (выше порога). | Геолокация работает в точном режиме. |
| **BB-40** | Эмуляция низкого заряда батареи (ниже порога). | Приложение переключается в режим экономии (неточная геопозиция). Проверить порог срабатывания (например, 15% или 20% - уточнить у разработчика по факту). |
| **BB-41** | Уведомление о смене режима геолокации. | Пользователь получает уведомление (toast/system alert) о том, что точность геопозиции снижена из-за заряда батареи. |
| **BB-42** | Влияние неточной геопозиции на маршрут. | Маршрут строится корректно, но точка старта может быть приблизительной (проверка адекватности работы алгоритма в этом режиме). |

## 4. Негативные сценарии и граничные условия (на основе пробелов в ТЗ)

| ID | Проверка | Ожидаемый результат |
|:---|:---|:---|
| **BB-43** | Отсутствие интернета при поиске. | Отображается понятное сообщение об ошибке сети, приложение не падает. |
| **BB-44** | Отказ в доступе к геолокации (если вдруг запретили в ОС). | Приложение запрашивает разрешение повторно или предлагает ввести местоположение вручную / показывает дефолтный город. |
| **BB-45** | Быстрое многократное нажатие на кнопку «Поиск». | Запрос отправляется один раз или debounce-механизм предотвращает дублирование запросов. |
| **BB-46** | Поиск в момент построения маршрута. | Корректная обработка конкуренции процессов (маршрут сбрасывается или поиск ставится в очередь). |
| **BB-47** | Изменение ориентации устройства во время построения маршрута. | Маршрут сохраняется, интерфейс корректно перестраивается под новый размер экрана. |
